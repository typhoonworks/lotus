# Lotus — Claude Code Guidelines

## Commits & Pull Requests

- **Do not** add `Co-Authored-By: Claude` or any Claude attribution to commits.
  The committer is the human user — always.
- **Do not** prefix branches or PR titles with `claude/` or `Claude:`.
- Use **Conventional Commits** for PR titles and commit messages:
  `<type>(<scope>): <description>`
  - Types: `feat`, `fix`, `refactor`, `perf`, `docs`, `test`, `chore`, `build`, `ci`
  - Scope is optional (e.g. `feat(dashboards): ...`)
  - Description: imperative, lowercase, no trailing period
- Branch names: `<type>/<short-desc>` (e.g. `fix/preload-dashboards`).

## IRON LAWS — STOP if violated

If code would violate ANY of these, you MUST:
1. STOP immediately
2. Show the problematic code
3. Show the correct pattern
4. Ask permission to apply the fix

**Ecto**:
1. NO `:float` for money → `:decimal` or `:integer` (cents)
2. Pin values with `^` in queries — never interpolate
3. Separate queries for `has_many`, JOIN for `belongs_to`

**Security**:
4. NO `String.to_atom(user_input)` — atom exhaustion attack
5. NO `raw/1` with untrusted content — XSS vulnerability

**OTP**:
6. NO process without runtime reason — processes are for concurrency/state/isolation
